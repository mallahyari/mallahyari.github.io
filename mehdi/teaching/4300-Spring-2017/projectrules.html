Ground rules concerning programming projects

  • Individual Programming Assignments: There will be a few individual programming assignments. You may not help another student or get help from any other person except the instructor and teaching assistants. You may not incorporate code from any other source, including our textbook, except as specifically authorized by the instructor. If you run across code that seems useful, you may study it, type it in and run it; but you must then erase it and write your own code. You may discuss ideas, but not code, with other students.
  • Group Projects: Same as for the individual exercises, except that you may help out other students in your group. However, each piece of code must be clearly labeled with the authorship
  • Group Term Project:

    Please sign up in teams of four on eLC. Each team will choose its own project in consultation from me. I especially encourage teams to develop some kind of e-commerce site. It needs to be a Web site, not a native app (this is a course in Web Programming, after all!)

    I will authorize smaller or larger teams only after we see whether the enrollment for this course is divisible by four or not. If a team member drops the course, the rest of the team will need to alter the project’s scope. I will merge teams only under extraordinary circumstances.

    As part of this project, your team will deliver several presentations to the class, describing your progress through the project. The topics should be roughly thus:

    1. Overview of the project
    2. Design artifacts: page mockups, class diagrams, and data model
    3. Technical and managerial challenges
    4. It’s done! Demo and lessons learned
