Project 3: Movie Reviews

In this project you will create a web application using Model-View-Controller (MVC) approach that involves movies, reviews and movie reviews. You must use the MySQL database called imdb2 (see below) to get the required information. The user name and password are given on wp420 in the directory /home/mehdi/cs4300/dbinfo.txt.

Turn In

These comments should make it easy to read and follow the logic of your code.  Also, your source code should be well formatted.   • A README.txt file providing the complete URL to use when invoking your application.

Project description:

  • Functionalities:

    Your application must provide the following functionalities:

    • View the titles of all movies of a selected genre
    • View information and reviews for a selected movie
    • Enter a new movie
    • Enter a review for a movie
    • Delete review(s) of a movie
    • Delete a movie

      • Please note that if a movie is deleted, all of its reviews must also be deleted.

  • You will need to create a database table, reviews, in the imdb2 database to store reviews for movies. The reviews table will need a foreign key linking it to the movies table. The reviews table must have the following fields: 
                    id INT(11)
                movie_id INT(11)
                review VARCHAR(256)
  • Your application must have an HTML file called index.html. This HTML file is the entry point (home page) of your application. The user should be able to use index.html to run different functionalities. Additionally, Your application should have a few HTML files, generated from FreeMarker templates, to provide the aforementioned functionalities. All HTML files except for index.html must be generated from FreeMarker templates.
  • Your program must have 4 packages as follows:
    1. edu.uga.cs4300.boundary: includes the Servlet class that handles all the requests and responses. The Servlet directly interacts with the controller class.
    2. edu.uga.cs4300.logiclayer: contains the Java class MovieLogicImpl. This class is the controller class for all the functionalities of the application. MovieLogicImpl class directly interacts with the Servlet and persistent layer classes.
    3. edu.uga.cs4300.persistlayer: includes the following Java files:
      • Define an abstract class called DbAccessConfiguration that includes all the variables (with proper values) required to connect to the database including DB_DRIVE_NAME, DB_CONNECTION_URL, DB_CONNECTION_USERNAME and DB_CONNECTION_PASSWORD.
      • Define a class called DbAccessImpl with the following methods: 
                public static Connection connect()
	
	public static ResultSet retrieve (String query)

	public static int create (String query)
	
	public static int update (String query)
	
	public static int delete (String query)
	
	public static void disconnect()
      • Define a class called MoviePersistImpl, which is the primary persistent (model) layer class. This class must explicitly communicate with the DbAccessImpl class to run the SQL queries.
    4. edu.uga.cs4300.objectlayer: includes the Movie and Review classes. These classes should be created according to the database tables, movies and reviews, respectively.
  • The application’s WAR archive must have a name of the form firstname_lastname_review.war and it must be deployed to our Wildfly Application Server on wp420 by copying the WAR file to the directory /opt/wildfly/standalone/deployments/project3.  Remember to set the read permissions of the WAR file to world-readable.  In order to set the permissions, you need to run the following Unix command after you copied the file into the project3 directory. 
    chmod a+x /opt/wildfly/standalone/deployments/project3/firstname_lastname_review.war
  • Your application must be accessible at the following URL:

    http://wp420.cs.uga.edu:8080/firstname_lastname_review/index.html

    (of course, substitute your name in the URL above).

    You have to use your username and password that were given in the class to able to connect to our server. The command to connect to our server is: 

    ssh your_username@wp420.cs.uga.edu

    Important Notice: Before you create (export) your .war file and deploy it on the server, please make sure you have changed the username and password to the correct ones for the database imdb2 on the server.

How to use MySQL database

  • You require to download and install MySQL available at here.
  • You can connect to the MySQL database using the default username/password (root/given_password): 
    mysql -uroot -pgiven_password
  • Create a database and call it imdb2.
  • Import the this imdb database to your newly created imdb2 database.
  • Check to make sure the imdb2 database has been populated successfully.
