Project 5: MyTunes, an MP3 Library Manager

Due:    July 29, 2016

In this project you will implement MyTunes, an improved version of your MP3 File Collection, developed in Project 4. The improvements include the addition of a Graphical User Interface (GUI), which enables convenient viewing of the information about the songs in a collection (author, title, etc.), sorting and searching the songs by author, title, etc., and playing the selected songs. As before, your program should be able to play an MP3 file and display some of the embedded tags in an MP3 file, such as title, author, album, and data, but all within a GUI interface. New features include sorting and searching of your MP3 files by any of the tags.

As before, this project involves dealing with mp3 files, and you must only use legally obtained mp3 files.

In this project, you are also required to extend your existing JavaDoc comments for all of your classes.

Submit your project directory to cs1302a on nike, a directory called Project5 (using the submit command):

  • All the source code, i.e. .java files (no need to include .class files).  Javadoc comments are required. Of course, regular “program logic” comments are required in your source code.  As usual, your source code should be well formatted.
  • A README.txt file telling us how to compile your program and how to execute it.

Project description:

  1. Your program must be in package edu.uga.cs1302.mytunes.

  2. Modify your implementation of the MP3File class to assign any undefined MP3 tags of author, title, and album to ”” (an empty string). Set an undefined date of an MP3 file to the year 1900. Also, add the following methods (you may add a suitable instance variable):

    int     getYear()
    void    setYear(int year)

  3. Implement a class called MP3Collection, which may be based on your MyMP3Files class developed in the previous project.  This time, the new class should represent your collection of MP3Files using Java's LinkedList. However, you must use only the List<E> interface to refer to the list. Subsequently, provide the following public methods:

               MP3Collection()

      Create an empty collection.

               MP3Collection(String directoryPathname)
      Create a collection containing files in the specified directory. The collection should include only files with the .mp3 extension.

    Object[][] getTableData()
      Return the data from the LinkedList of MP3 files as an array, where each row represents the author, title, album, and year of one MP3 file. The number of columns should be 4, and the number of rows should be equal to the number of MP3 files in the List holding the MP3 files.

    int        size()
      Return the size of the collection.

    MP3File    getFile(int index) throws IndexOutOfBoundsException
      Return an MP3File at the given index position in the collection.  Throw an IndexOutOfBoundsException exception if the index value is out of bounds.

    void       startPlay(int index) throws IndexOutOfBoundsException
      Start playing a file (MP3 track) at a given index position. Throw an IndexOutOfBoundsException exception if the index value is out of bounds.

    void       stopPlay()
      Stop the currently playing file.

  4. Create a Graphical User Interface (GUI) program, called MyTunes, serving as a manager of your collection. The MyTunes program should be implemented using a JFrame with suitable buttons, a text field, and other components, as needed. IMPORATANT: you must use JTable component to represent the songs in your MP3 collection.

    The JTable component should be initialized with the data obtained from the getTableData method in the MP3Collection class. The columns should be labeled as ”Author”, ”Title”, ”Album”, and ”Year”.

    The MyTunes program should provide GUI-like capabilities. In particular, it should be possible to:

    • View the MP3 files; scrolling up/down, if only a portion of the files fits within the JTable view should also be possible. You should create a JScrollPane for the JTable. The usual functionality of a JTable, such as resizing and repositioning the columns should be available, as well.

    • Sort any of the four columns (author, title, etc.) by clicking on the column header; repeated clicks on the same column should reverse the sorting order.  You should create a TableRowSorter for the JTable.

    • Filter the shown MP3 files by providing a substring (it should be present in any of the columns). The string to be used in filtering should be entered into a JTextBox, and then pressing the Enter key should activate the filter.  You should create a RowFilter.regexFilter (using a static method) for the sorter used in the JTable.

    • Cancel an existing filter by pressing a suitable button.

    • Play the selected MP3 file (the selection should be within the JTable showing the MP3 files). Pressing the Play button should first invoke the stopPlay() method (in case a song is playing when the user selects a different song to play) and then invoke the startPlay() method of the MP3Collection. A slightly modified version of the MP3Player class is on nike.

    • Stop the playback of an MP3 file by pressing a suitable button. Pressing the Stop button should invoke the stopPlay method of the MP3Collection.

    • Closing the MyTunes window (JFrame) should terminate the program. You may add a Quit button, if you like.

    The three buttons (Play, Stop, Cancel filter) and the text box used for filtering should be located at the top of the frame, and the JTable should occupy the whole lower part of the JFrame. You should use the Box layout (layout manager).

    The MyTunes program should either accept a directory with the MP3 files as a command line argument, or, if not provided, use the directory called “Music” in the current directory.

  5. You should provide Javadoc comments in your program. However, no JUnit testing is required in this project.

  6. Additional information will be provided in class, on eLC, and on the class Web site.
