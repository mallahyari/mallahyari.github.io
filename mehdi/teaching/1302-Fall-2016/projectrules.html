Ground rules concerning programming projects

  • All programming projects will be done on Linux, using Sun's Java, version 1.8.
  • Programming projects are due at midnight of the due date.
  • There are no scheduled labs for this class (unlike CSCI 1301). Therefore, all programming projects will be done by students individually, on their own time.
  • All programming projects must be submitted electronically (using the submit utility) on our departmental server nike. The submitted programs must compile and run on nike, using Sun's Java, version 1.8, available on nike. It is your responsibility to upload your program files to the server, compile the program and test it to make sure it runs correctly.
  • It is expected that you will be developing your projects on nike, since learning to use Unix/Linux is one of the objectives of this course. However, if you decide to develop your projects on a different system (MS Windows, Mac OS X, or a different Unix/Linux system), even using an IDE (Eclipse, NetBeans), you must transfer your project to nike, make sure that it compiles properly using the Java compiler available on nike, test it, and then submit it for grading.
  • You should start early on programming assignments and I will remind everyone of this several times. One suggestion: if you think that you can do a program in 2 days and you have 1 week to do it, do it during the first two days and enjoy the rest of the week for other things.
  • If you are sick and have appropriate documentation, always let me know as soon as you get sick. It will be to your disadvantage to tell me after the fact that you were sick; you may not be excused from the regular due date.
